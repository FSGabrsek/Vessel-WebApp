Vessel

Vessel is a webapp that makes it easy to track all media you're watching, reading, or planning to watch in the future. On Vessel you can add pieces of media to your personal list, organise them by type, provide them with a description, track your progress by episode or minute and leave a review to share with your friends when you're done.

Datamodel

Vessel uses a simple datamodel to store the media you and others watch.

MediaSoul

MediaSouls are the core of Vessel's data structure. Souls represent an individual user's interactions with a piece of media. It stores the current progress as a number, the last time the progress was updated as a date and based on the progress in relation to the media's total length it stores a calculated measure 'Status'. This indicates whether the media is 'Finished', 'In Progress' or 'Not Watched'.

Souls must belong to a MediaVessel via a 0 or many to 1 relation and Users watch souls via a 1 to 0 or many relation. Souls are graded by reviews, which they store via a 1 to 0 or 1 relation.

Review

Users can rate media on Vessel by leaving reviews. Reviews have a rating (either +, - or ~) and a content field for a more detailed description. Internally reviews also store a date to track when reviews are created and updated.

MediaVessel

Whereas souls are an individual user's expirience with a piece of media, vessels represent the media itself. Vessels store whether they are a series, a film or literature as 'Type'. A vessel also stores a title, synopsis and it's length. For vessels that have not been completely released, length is split into a current length and an eventual final length. Whether the media is upcoming, currently airing or finished is stored in as 'status'. Finally, the vessel stores its release date, and how often new parts are released as dates.

Only the original creator of a mediavessel is allowed to edit its data, so the vessel's owner is stored in a 1 to 0 or many relation from a user to vessels.

User

Users of vessel are represented as user entities. A user's username and password are stored in addition to an email adress for authentication and a date of birth.

Users can follow other users to see their vessels. Following is represented as a 0 or many relation from the user entity to itself.

User Stories

Add media to my list

As a user I want to create a vessel with various details about a piece of media and add it to my list so I can later view or edit the details

View progress

As a user I want to track and update the progress on every vessel on my list

Review media

As a user I wasnt to write reviews on vessels on my list that I can later view or edit

Follow friends

As a user I want to follow other users so I can view their lists and add their vessels to my list
